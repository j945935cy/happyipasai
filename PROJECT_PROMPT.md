# PROJECT_PROMPT.md

## 一、專案名稱
Happy iPAS AI

---

## 二、專案目標

本專案目標是建立一個「AI / 程式設計 / iPAS 證照學習網站」，並逐步將 Blogger（happyipasai.blogspot.com）上的內容搬移到 GitHub Pages。

本網站需具備以下特性：

- 作為學習入口網站（入口導向）
- 提供教學文章（閱讀導向）
- 提供課程教材（章節導向）
- 結構清楚，方便持續擴充
- 可由 Codex 持續維護與生成內容

---

## 三、網站整體設計風格（核心）

本網站採用「三種頁面角色混合設計」：

### 1. 首頁（入口頁）
風格參考：
- Notion
- freeCodeCamp

用途：
- 導覽入口
- 提供分類與快速連結

設計要求：
- 卡片式 UI
- 分類清楚
- 不放過長文章
- 重點在「去哪裡」

應包含區塊：
- Hero 主視覺
- 網站簡介
- 初級課程入口
- 中級課程入口
- 免費測驗入口
- 最新文章
- 精選教材
- FAQ

---

### 2. 文章頁（閱讀頁）
風格參考：
- Medium

用途：
- 顯示單篇教學文章或觀念文

設計要求：
- 閱讀優先
- 大標題
- 大留白
- 行距舒適
- 內容置中
- 避免干擾

應包含：
- 標題
- 發布日期
- 分類
- 文章內容
- 返回文章列表
- 返回首頁

---

### 3. 教材頁（章節頁）
風格參考：
- GitBook
- Docusaurus

用途：
- 顯示課程或教材

設計要求：
- 左側章節目錄
- 右側內容區
- 支援章節切換
- 適合長時間學習

應包含：
- 側邊欄（章節）
- 內容區
- 上一章 / 下一章
- 返回首頁

---

## 四、技術要求

- 使用 HTML + CSS + JavaScript
- 不使用 React / Vue / 框架
- 可直接部署到 GitHub Pages
- 使用 UTF-8 編碼
- 檔案結構清楚、可維護
- CSS 統一管理
- 支援基本響應式（手機可閱讀）

## 六、設計原則（非常重要）

### 整體風格
- 清爽
- 教學導向
- 專業但不複雜
- 適合長時間閱讀

### UI 原則
- 留白要足夠
- 不要過度顏色
- 不要過多動畫
- 卡片風格統一
- 按鈕風格一致

### UX 原則
- 使用者可以快速找到內容
- 層級清楚
- 導覽簡單

---

## 七、內容處理原則

本專案將逐步從 Blogger 搬移內容，請遵守：

- 每篇文章為獨立 HTML
- 保留標題與段落結構
- 清理不必要的 HTML 標籤
- 統一版型
- 外部連結整理為按鈕或清單
- 避免直接貼 Blogger 原始格式

---

## 八、Codex 工作方式（重要）

Codex 必須遵守以下流程：

1. 先建立版型（不要一開始塞大量內容）
2. 使用假資料建立完整頁面
3. 確保三種頁面風格一致
4. 再逐步加入實際內容
5. 每次修改要保持整體風格一致

---

## 九、禁止事項

- 不要直接複製 Notion / Medium / GitBook 原始碼
- 不要使用大型前端框架
- 不要讓版面過於複雜
- 不要讓頁面風格不一致
- 不要將所有內容塞在首頁

---

## 十、輸出要求

每次產出必須：

- 可直接開啟（HTML 可執行）
- 結構清楚
- CSS 可重用
- JS 簡單明確
- 頁面風格一致

---

## 十一、優先順序（開發順序）

1. 建立 index.html（首頁）
2. 建立 article.html（文章頁）
3. 建立 course.html（教材頁）
4. 完成 CSS
5. 加入假資料測試
6. 再開始搬 Blogger 內容

---

## 十二、設計理念總結

本網站不是單純部落格，而是：

- 學習入口（Homepage）
- 閱讀平台（Article）
- 教材系統（Course）

三者整合的一個教學網站。

---

## 十三、最重要原則（一句話）

首頁導流、文章閱讀、教材學習，各自分工，但風格一致。
